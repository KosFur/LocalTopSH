# Knowledge Base Documents

Положите сюда документы для базы знаний.

## Поддерживаемые форматы:
- `.docx` - Microsoft Word
- `.doc` - Microsoft Word (старый формат)
- `.pdf` - PDF документы
- `.txt` - Текстовые файлы
- `.md` - Markdown

## Структура папок

Создавайте папки для категорий:

```
knowledge_docs/
├── Возвраты/
│   ├── Инструкция по возвратам.docx
│   └── Политика возврата денег.pdf
├── Доставка/
│   ├── Сроки доставки.docx
│   └── Работа с курьерами.pdf
├── FAQ/
│   └── Частые вопросы.docx
└── Общее/
    └── Корпоративные стандарты.pdf
```

Названия папок будут использованы как **категории** для фильтрации.

## Загрузка в базу знаний

После добавления документов:

```bash
# Первичная загрузка
npm run kb:ingest

# Или из конкретной папки
npm run kb:ingest ./knowledge_docs

# Полная перезагрузка (удалить старое + загрузить новое)
npm run kb:ingest -- --reset

# Проверить статус базы
npm run kb:status
```

## Требования

1. **Qdrant** должен быть запущен:
   ```bash
   docker compose up qdrant -d
   ```

2. **API для эмбеддингов** должен поддерживать `/v1/embeddings` (OpenAI-совместимый)

## Советы

- Разбивайте большие документы на логические части
- Используйте понятные названия файлов
- Добавляйте заголовки в начало документов
- Группируйте по категориям для удобного поиска
